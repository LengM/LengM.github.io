给定一个数字，计算出从0开始到该数字内所有数字之和。

  • 传入-5,输出
    -5<0
  • 传入0,输出
    0=0
  • 传入6,输出
    0+1+2+3+4+5+6=21

我的解决方法：

一开始自己选择循环变量从0开始，也是陷入了输出结果的误区（从0开始，在其后面拼接+号），在拼接字符串时最后就会多出一个+号，需要手动截取字符串，变得很麻烦
优化后更好的解决方法：

优化之后，我们将返回的字符串默认值为0，这样我们的循环变量从1开始，只需要在其前面拼接+号即可，这样就会避免多出+号的问题了。

封装类型，引用和实例，题一

  • 以下代码输出结果是什么？

  • 一开始我以为输出结果是null null 42，但是实际结果是null 42 42

    解析：

    • 首先Two类中的x类型是封装类，默认值为null，所以第一次输出的t.x是null
    • 注意t2，是通过fix方法获取的，而fix方法返回的实例实际上就是传入的t引用指向的实例
      那么就有以下结论：
      • t,tt,t2三个引用指向同一个实例对象
      • t.x = t2.x = tt.x = 42
    • 最后输出null 42 42
